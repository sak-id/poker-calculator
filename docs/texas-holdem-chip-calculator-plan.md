# テキサスホールデム用 チップ計算Webアプリ提案

## 1. ゴールと前提
- **目的**: 対面プレイ中に1人の記録係が、ベット/レイズ/オールイン/ポット配分を正確に記録・計算できること。
- **対象**: テキサスホールデム（ライブ対面、カードは実物使用）。
- **非対象（初期版）**: オンライン対戦連携、ハンド履歴解析、カード画像認識。

## 2. 想定ユースケース
1. ゲーム開始時に参加者と初期スタックを入力。
2. 各ストリート（Preflop/Flop/Turn/River）で、
   - プレイヤーのアクション（fold/call/check/bet/raise/all-in）を記録。
   - ベット額・レイズ額は**電卓UI**で入力。
3. ショーダウン後に勝者（複数可）を選択し、ポットを自動配分。
4. 次ハンドへ進むと、スタックが更新された状態で継続。

## 3. 必須機能（MVP）

### 3.1 テーブル管理
- プレイヤー追加・削除・並び順変更（席順管理）。
- 初期チップ設定（全員同額/個別入力）。
- ブラインド設定（SB/BB、アンティ任意）。

### 3.2 ハンド進行管理
- ディーラーボタン位置の自動ローテーション。
- 現在アクションプレイヤーの表示。
- ストリート進行（Preflop→Flop→Turn→River→Showdown）。
- 最小レイズ額の自動計算。

### 3.3 金額入力（電卓形式）
- テンキー: 0-9 / 00 / C / ← / + / - / × / ÷ / =。
- ショートカット: 1/2 Pot, 2/3 Pot, Pot, All-in。
- 「最終入力値をそのままアクションに適用」ボタン（Bet/Raise/Call）。
- バリデーション:
  - 現在スタックを超えない
  - 最小レイズ未満の扱い（オールイン時の例外あり）
  - 端数ルール（整数チップ前提）

### 3.4 ポット・サイドポット計算
- メインポットと複数サイドポットの自動生成。
- ショーダウン時に勝者をポット単位で選択。
- 同着時の均等配分（余りチップ配分ルールを固定化）。

### 3.5 監査・戻る操作
- 直近操作Undo（最低1手、推奨10手）。
- 1ハンドのアクションログ表示。
- 明示的な「確定」操作を置き誤タップを防止。

## 4. UI/UX提案
- **レイアウト**（タブレット横向き想定）
  - 左: プレイヤー一覧（スタック、現在投入額、状態）
  - 中央: ポット情報（Main/Side）とハンド進行
  - 右: 電卓パネル＋アクションボタン
- 色分け:
  - アクション待ち: 強調色
  - Fold済: グレーアウト
  - All-in: バッジ表示
- 操作ミス対策:
  - 大きいボタン
  - 危険操作（ハンド確定・リセット）に確認ダイアログ

## 5. ドメインルール実装のポイント
- `committedThisRound`: ストリート内投入額
- `committedThisHand`: ハンド全体投入額
- `currentBet`: そのストリートで追いつくべき額
- `lastFullRaiseSize`: 最小レイズ判定に必要
- オールインでフルレイズ未満の場合、アクション再オープンの扱いに注意（ルール設定で切替可能にするのが安全）。

## 6. おすすめ技術スタック
- **フロントエンド**: React + TypeScript + Vite
- **UI**: Tailwind CSS + Headless UI（または shadcn/ui）
- **状態管理**: Zustand（ゲーム状態を1箇所で管理）
- **計算ロジック**:
  - 純粋関数として分離（`domain/`）
  - UI層から独立させてテスト容易性を確保
- **永続化**: LocalStorage（MVP）
- **将来拡張**: IndexedDB / Firebase / Supabase

## 7. データモデル（例）
```ts
interface Player {
  id: string;
  name: string;
  seat: number;
  stack: number;
  status: 'active' | 'folded' | 'allin' | 'out';
  committedThisRound: number;
  committedThisHand: number;
}

interface Pot {
  id: string;
  amount: number;
  eligiblePlayerIds: string[];
}

interface HandState {
  handNo: number;
  street: 'preflop' | 'flop' | 'turn' | 'river' | 'showdown';
  dealerSeat: number;
  sb: number;
  bb: number;
  currentBet: number;
  lastFullRaiseSize: number;
  mainPot: number;
  sidePots: Pot[];
  actionLog: Action[];
}
```

## 8. 実装ステップ（推奨）
1. **ドメインロジック先行**
   - ベット妥当性判定
   - ポット/サイドポット生成
   - 配分計算
2. **電卓コンポーネント作成**
   - 入力式表示 + 計算結果
   - クイックベットボタン
3. **ハンド進行UI実装**
   - 現在プレイヤーの遷移
   - ストリート遷移
4. **ログ/Undo/確定フロー追加**
5. **E2Eテスト**（代表シナリオ）
   - 2人ヘッズアップ
   - 6-maxで複数オールイン
   - 同着分配

## 9. テスト戦略
- 単体テスト（最重要）:
  - 最小レイズ判定
  - オールイン例外
  - サイドポット分解
  - 端数処理
- 統合テスト:
  - 1ハンド通しの状態遷移
- E2E:
  - 実プレイ想定の操作フロー（Playwright）

## 10. 先に決めるべき運用ルール
- 同着時の余りチップを誰に渡すか（ボタン左優先など）。
- トーナメント想定かキャッシュ想定か。
- ブラインドレベルアップを管理するか。
- Rebuy/Add-onを扱うか。

## 11. 追加のおすすめ
- **ミス防止モード**: 金額確定前に「声出し確認」用の大きな確認表示。
- **時短モード**: よく使うベットサイズをプリセット。
- **表示最適化**: スマホ縦画面向け簡易モード。
